# Описание создания тестов для ИИ-агентов

## Семантические правила
- указания вида "создать тест" и подобные по-умолчанию должны создавать unit-тесты
- unit-тесты должны проверять только код указанного класса или функции

## Тестовый фреймворк
- для создания тестов должен использоваться фреймворк jest
- должна использоваться функция jest describe() для смысловой группировки тестов
- возможна вложенность одного describe() блока в другой
- должна использоваться функция jest it() для запуска отдельного теста
- сама проверка должна проводиться функцией jest expect()

## Использование параметров
- если класс ожидает какие-либо зависимости в конструкторе, то вместо этих зависимостей нужно использовать их мок-версии
- мок-версии должны содержать только ту функциональность, которую использует тестируемая логика
- мок-версии должны создаваться для параметров функции, если тест нужно создать для функции
- мок-версии стоит создавать в отдельных файлах

## Расположение файлов
- тест должен создаваться в отдельном файле
- файл должен иметь то же название, что и класс или функция, подлежащий тестированию, но расширение должен иметь `.spec.ts`
- если для теста нужны дополнительные данные (фикстуры, мок-версии), то их следует создавать в отдельной папке. тогда
  для теста следует создать папку `имя_файла_test`, и там расположить `.spec.ts` файл, а также сопутствующие 
  файлы фикстур и/или мок-сервисов

## Реализация тест-логики
- тестирование нужно разбивать на небольшие блоки describe() и it() функций
- переменные должны называться по смысловому их содержимому
- код должен быть лаконичным
